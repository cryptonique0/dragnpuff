# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in House of the DragNs, please email security@dragnpuff.xyz instead of using the issue tracker.

### What to include
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

### Timeline
- We will acknowledge your report within 48 hours
- We will provide an estimated timeline for a fix
- We ask that you do not publicly disclose the issue until we've had time to address it

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Security Best Practices

When using House of the DragNs smart contracts:
- Always use the latest version
- Never share your private keys
- Verify contract addresses before interacting
- Use hardware wallets for large transactions
- Audit contracts before deployment

## Dependencies

We regularly update dependencies to patch known vulnerabilities. Run `npm audit` to check for vulnerabilities in your dependencies.
