# Contributing to House of the DragNs

Thank you for your interest in contributing to House of the DragNs! This document provides guidelines and instructions for contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/dragnpuff.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Push to your fork and submit a pull request

## Development Setup

### Prerequisites
- Node.js 18+
- Git
- Basic understanding of Solidity and blockchain concepts

### Installation
```bash
npm install
```

### Running Tests
```bash
npm test
```

## Code Style

- Follow Solidity conventions for contracts
- Use meaningful variable and function names
- Add comments for complex logic
- Include JSDoc comments for JavaScript functions

## Commit Messages

- Use clear, descriptive commit messages
- Use imperative mood ("add feature" not "added feature")
- Keep commits focused on a single change

## Pull Request Process

1. Update documentation for any new features
2. Add tests for new functionality
3. Ensure all tests pass
4. Provide a clear description of changes in the PR

## Smart Contract Guidelines

- Use OpenZeppelin contracts where possible
- Follow Solidity best practices
- Include comprehensive comments
- Use events for important state changes

## Questions?

Open an issue on GitHub if you have questions.
